# 判定手册

判定是整个审核的核心环节。这份手册定义两层检测怎么协作、违规锚点怎么用、尺度松紧怎么拿捏。尺度部分基于 38 篇平台实际放行的真实口播稿校准过：字面从严会造成系统性误报，让用户不再信任"必改"结论。

## 两层检测怎么协作

**词面层（scan.py）**负责召回：它快、稳定、可复现，能把值得看的位置标出来，但它不懂语境。
**语义层（你）**负责结论：每个候选放回原文判断，同时独立扫一遍全文找"没有关键词的风险"。

两层的分工铁律：

- scan 命中 → 必须逐条判定，不许因为"看起来没事"批量清空；
- scan 零命中 → 仍要按适用规则通读全文做语义检查，重点看：收益暗示、功效暗示、身份冒充、导流意图、伪造人设——这些常常整段违规但不含任何关键词；
- 判定顺序：先高风险候选，再低风险候选，最后全文语义扫描。

## 违规锚点：判"违规"要有抓手

"违规"不是感觉，是命中具体锚点。每条规则的锚点写在规则文档的"怎么判"里，通用模式是：

**违规 = 越线表达 + 危险绑定**

- 收益类：确定性用词（保证/稳/肯定/必）**绑定**金额或期限；或收益卖点**绑定**付费转化（买课/进群/下单）；
- 功效类：固定期限**绑定**确定效果（"28天根治"）；
- 人设类：财富展示**绑定**转化引导且无法证实；
- 导流类：站外平台/联系方式**绑定**行动指令（加/搜/扫/进）。核心一问：**这句话有没有给观众布置"平台之外"的动作？**布置了（哪怕是"自己去应用商店搜"这种由观众完成的动作）就越线；只陈述自己的使用经历、不布置动作的种草式介绍不越线。指向目标平台自家生态产品（如抖音场景下的剪映、抖音商城）不算站外。

只有越线表达没有危险绑定的（比如提到收入数字但没承诺、没转化），**不构成违规**。

## 校准尺度（真实样本教训）

以下表达在 38 篇实际放行的真实稿件中大量出现，判定时给**风险提示**而不是"必改"：

| 表达类型 | 例子 | 判法 |
|---|---|---|
| 个人经历数字 | "我上个月赚了3万" | 提示：数字宜可证、别叠加确定性用词；不拦 |
| 泛化鼓励 | "你也可以试试" | 无确定性承诺、无付费绑定就不拦 |
| 学员案例陈述 | "我有个学员做到了X" | 同上；绑定买课转化才升级 |
| 有边界的主观体验 | "我用过最顺手的" | 限定了"我用过"，不是产品绝对化 |
| 行业隐喻 | "账号诊断""效率翻倍" | 普通语境不触发医疗/金融规则 |

反过来，以下情况**必须拦**，不因用户不满而松口：

- 确定性承诺绑定金额/期限（"保证三十天回本"）；
- 收益/功效卖点直接绑定付费转化；
- 无资质给出医疗/金融专业结论并引导行动；
- 伪造迹象（摆拍冒充真实、虚构身份获利）；
- 监管红线（虚假广告、荐股、处方药推广等——见行业文档）。

## 商业属性判定

很多规则只在商业场景生效。判断内容有没有商业属性看**转化意图**，不看用户自我声明：

- 有：带货、卖课、接广、返佣、引流到私域成交、为自己的付费服务做铺垫；
- 无：纯经验分享、知识科普、生活记录，无任何转化承接；
- **不明：按"有"处理**。宁可多套用商业规则（最多严一点），不可漏套用（漏判）。

## 行业判定

看内容实际讲了什么，不看账号定位：出现疾病/治疗/药品/保健品/医美 → 叠加医疗文档；出现股票/基金/贷款/保险/收益率/理财课 → 叠加金融文档。行业词只是隐喻时（"治愈系风景""财务自由的心态"）不触发，但在判定理由里写一句为什么不触发。

## 个人规则怎么参与判定

`data/my-rules.md` 的四个区各有各的判法，用户自己沉淀的规则和官方规则**同等地位**：

| 区 | 判法 |
|---|---|
| 黑名单·语境判（默认） | scan 标出候选 → 像官方规则候选一样结合上下文判定，用户的"为什么"是判定输入 |
| 黑名单·见词即报 | 用户确认过的零容忍词，命中直接按其级别进结论，报告注明"你自定的零容忍词" |
| 语义规则（`MY-` 编号） | 逐条按锚点检查全文——处理方式与官方规则完全相同：命中给原文引用、按其级别定档、用其"怎么改"；报告引用 `MY-` 编号和来源 |
| 语义笔记 | 判定的背景知识，优先级高于一般直觉 |

边界：个人规则只能**加严或降噪**——白名单和笔记不能豁免官方规则拦下的真实违规，个人规则的"见词即报"也不能低于官方规则对同一表达的定档。

## 判定输出纪律

- 每个风险项必须引用原文（可核对），标规则编号（官方规则号或 `MY-` 个人规则号），给具体改法；
- 三档结论：**必改**（命中锚点）/ **建议改**（有风险因素但未越线）/ **仅提示**（辟谣、写法建议）；
- 缺信息就标"无法判定"+缺什么，不硬判；
- 任何用户配置都不能豁免本手册"必须拦"清单。

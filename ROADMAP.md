# Roadmap

需求和优先级由真实使用反馈驱动：想要什么、哪里不准，开 issue 或在社群里说一声。

## V1.0（当前）

- 抖音 / 小红书 / 微信视频号三平台
- 口播稿、文章、图文文案、字幕、标题等文本审核
- 两层检测（词面预检 + 语义判定），38 篇真实样本校准的判定尺度
- 保意修复 + 辟谣提示
- 个人沉淀：词库黑白名单、语义笔记、被罚案例复盘、审核历史

## V1.1（下一步）

- **封面与画面检查清单**：利用模型视觉能力审封面文字、画面元素、水印、二维码
- **对比评测基线**：建立 with/without skill 的配对评测（真实场景语料、盲评打分），把"有效"从盲测级证据升级为可复现的对比数据
- **快手平台规则包**（来源采集 + 规则提炼）
- **直播带货专项**：直播话术、商品卡、福利互动、价格表述的专项规则
- **海外内容适配指南**：X/YouTube 内容转国内发布的完整检查（来源授权、语境转换、平台名称处理）

## V1.2

- 更多行业规则包：教育培训、母婴、美妆个护（优先级按用户反馈排序）
- 审核历史的聚合盘点（"我最常踩哪类规则"）
- 个人规则的导出/分享格式（团队共用一份行业词库）

## V2.x

- 社区规则共建机制成熟化：贡献模板、脱敏案例库、规则评审流程
- 规则订阅更新：CHANGELOG 驱动的规则版本管理
- 多账号档案支持（代运营场景）

## 长期原则

- 规则必须锚定可查证来源，社区经验单独分层，不与官方规则混淆；
- 判定尺度持续用真实放行/处罚样本校准，误报和漏报同样是 bug；
- **规则保鲜节奏**：每季度复核来源链接与核验日期，平台重大规则变更随时热更；每次规则变更过上游回归后才发布，记入 CHANGELOG；
- 用户沉淀永远本地、可读、可迁移。
